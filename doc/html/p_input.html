p_input module¶

Functions generating input for brian2 model neurons: spike times for SpikeGenerator and step and ramp currents. (generate_current partly reuses code from http://neuronaldynamics.epfl.ch) [call-time-model]

p_input.generate_current(t_start, sim_time_ms, unit_time, amplitude_start, amplitude_end=None, n_neurons=1, offset_range=0.0 * amp, offset_noise=None, wnoise_cv=0, wnoise_dt=1, white_noise=None, b_power_ramp=None)[source]¶

Generate input current as numpy.array. t_start and amplitude_start must be ints/Quantities or lists with the same number of elements. Each element corresponds to one step current or ramp current in a sequence of currents. Elements must be ordered ascending in time. Returned array has n_neurons. Columns are identical if offset_range==0.

Parameters

  • t_start (int or float or list) – integer or float or list of integers/floats with start times of current

  • sim_time_ms (int or float) – duration of simulation in milliseconds

  • unit_time (b2.units.fundamentalunits.Quantity [unit=sec]) – brian2 unit of time step (dt between successive current values), e.g. 0.1*b2.ms. Ideally, to have the same time step as the simulation, set this to b2.defaultclock.dt. Can be set to longer, esp. without ramps.

  • amplitude_start (b2.units.fundamentalunits.Quantity [unit=amp] or list) – quantity or list of quantities (unit: ampere) of amplitudes at t_start.

  • amplitude_end (b2.units.fundamentalunits.Quantity [unit=amp] or list or None) – [default=None] quantity or list of quantities of amplitudes at t_end. Generated current ramps from amplitude_start at t_start to amplitude_end at t_end. If amplitude_end is None, [] or empty np.array, it is set to amplitude_start, i.e. only constant step currents are generated.

  • n_neurons (int) – [default=1] number of neurons == number of columns in the returned current array

  • offset_range (b2.units.fundamentalunits.Quantity [unit=amp]) – [default=0] the given current amplitude gets offset by a random value within offset_range. Random values are normally distributed and can exceed these limits! (code for uniformly distributed random offsets commented out below)

  • offset_noise (numpy.ndarray or list) – [default=None] numpy array of random numbers used in randomly offsetting current amplitudes. Empty list: no offsets

  • wnoise_cv (float) – [default=0] coefficient of variation for white noise added to full current

  • wnoise_dt (float or int) – time in milliseconds between successive random changes in input current according to white_noise

  • white_noise (list) – [default=None] list of numpy arrays (n_neurons) of random numbers used for additive white noise

  • b_power_ramp (list or None) – [default=None] None or list of bools of len(t_start). where True or 1, ramp currents increase with power 2 instead of a linear ramp

Returns

current: numpy.array to be used as input_current to model neurons. Convert to b2.TimedArray before use, using p_input.timedarray_from_current()

Return type

brian2.units.fundamentalunits.Quantity [unit=amp]

p_input.load_spiketimes_for_gen(path_to_spiketimes, neuron_nrs=None, trace_nrs=None, population_nrs=None, recording_offsets_ms=None, b_single_population=False)[source]¶

Load spiketimes from recordings of specific neurons to use as input for SpikegeneratorGroups.

Parameters

  • path_to_spiketimes (str) – path to directory containing spiketimes pickle files (.pkl)

  • neuron_nrs ([int] or None) – [default=None] list of integer indices to neurons from which spiketimes should be loaded. 1-based indexing of alphanumerically sorted .pkl filenames in path_to_spiketimes. None: load from all files

  • trace_nrs ([[int]] or None) – [default=None] list of list(s) of integer indices to traces of the respective neuron (one sublist per neuron) from which spiketimes should be loaded. 1-based indexing. None: load from all traces

  • population_nrs ([int] or None) – [default=None] list of integer identifiers for grouping neurons into populations (used for spikegenerator populations), one integer per neuron. Must be in ascending order. None: either all neurons belong to the same population, or each neuron is part of its own population, depending on b_single_population

  • recording_offsets_ms ([float] or None) – [default=None] list of offsets in milliseconds, which will be added to spiketimes to account for differential alignment. one offset per neuron. if None, offset will be zero for all neurons

  • b_single_population (bool) – [default=False] if True, all neurons will be put in a single sublist, i.e. will belong to a single spikegenerator population when passed to run_simulation(). Otherwise each .pkl file will (e.g. recorded neuron) will make up a seperate sublist / spikegenerator population. If population_nrs is not None, this parameter is ignored

Returns

input_spiketimes_ms: list of list(s) of list(s) of spike times in milliseconds. one sublist per generator population (corresponds to model neuron population), one subsublist per trace, i.e. generator neuron. To be passed to run_simulation() for creation of spikegenerator groups

Return type

[[[float]]]

p_input.setup_currents(config, offset_noise, white_noise, log_name, sim_time_ms=None, b_gen_curr_per_pop=None, I_input_prev=None)[source]¶

Generate and return time series of currents to be used as input onto model neurons, based on specifications in config (e.g. loaded from .json config file).

Parameters

  • config (dict) – dictionary of model parameters as loaded from .json configuration file

  • offset_noise (list) – list of numpy arrays of random numbers used in randomly offsetting current amplitudes. one array per population. empty list instead of array where no numbers were generated (one [] per pop)

  • white_noise (list) – list of list(s) of numpy arrays of random numbers used for additive noise in current generation. one sublist per population, one array per neuron. list of Nones if no numbers were generated (one None per pop)

  • log_name (str) – name of logger object (standard python logging) created earlier, e.g. through p_io.set_up_loggers()

  • sim_time_ms (int or None) – [default=None] duration of simulation in milliseconds. if None: load from config file

  • b_gen_curr_per_pop (list or None) – [default=None] list of one bool per population. if true, generate currents only for the respective population and overwrite I_input_prev for those. if None, generate currents for all populations.

  • I_input_prev (brian2.TimedArray or None) – [default=None] output of this function from the previous run. Supply if b_gen_curr_per_pop.

Returns

  • I_input: brian2.TimedArray containing all generated currents. (May be used in ode as Ie)

  • log_message: string containing message to be printed/logged

Return type

  • I_input: brian2.input.timedarray.TimedArray

  • log_message: string

p_input.timedarray_from_current(currents, unit_time, name=None, pad_to_length=None)[source]¶

Bundle multiple current arrays returned by p_input.generate_current() into a brian2.TimedArray.

Parameters

  • currents (b2.units.fundamentalunits.Quantity [unit=amp] or list) – (list of) numpy.array(s) returned by p_input.generate_current(), one for each neuron/-group

  • unit_time (b2.units.fundamentalunits.Quantity [unit=sec]) – brian2 unit of time step (dt between successive current values), e.g. 0.1*b2.ms. Ideally, to have the same time step as the simulation, set this to b2.defaultclock.dt (might not work otherwise).

  • name (str or None) – [default: None] sets name parameter for returned brian2.TimedArray

  • pad_to_length (int or None) – [default=None] currents will be padded with zeros to this length (i.e. number of time steps)

Returns

timed_array_current: brian2.TimedArray containing all passed currents

Return type

brian2.input.timedarray.TimedArray
